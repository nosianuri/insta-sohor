1. JavaScript is a Programming Language for the Web. It used both on the
client-side and server-side that allows to make web pages interactive. It runs
inside the web browser or on the server and can do a lot of things. It's also
used onthe back-end to store and share data.It is to validate data given by the
user in the form fields.

2. The event loop is the secret by which JavaScript gives us an illusion of
being multithreaded even though it is single-threaded. It is a constantly
running process that monitors both the callback queue and the call stack.
When a function is finished, it is popped from the stack. A function that
takes a long time to complete is called a blocking function. A blocking
function blocks all the interactions on the webpage.

3. The difference of LocalStorage and SessionStorage:
# LocalStorage persists the storage across browser sessions. But sessionStorage
is cleared when the page session ends.
# LocalStorage is no expiration. But sessionStorage creates a new session and
new storage.
# LocalStorage has 5 mb limit. But SessionStorage has 5-10 mb limit.
# LocalStorage is not session based, need to be deleted via JS or manually. But
sessionStorage is session based and working per window or tab.
